This example demonstrates how to use a smart card to digitally sign data and manage user certificates, it  implements the PKI API in the Security and Trust Services API 1.0 defined in JSR 177.

1. About this example

This example implements SATSA-PKI APIs. It demonstrates how to use a smart card to digitally sign data and manage user certificates.  This example currently does not come with the certificate, so the user can not run it on the device or SDK.

2. Pre-requisites

The example makes use of the standard MIDP application framework, security service and PKI API. This example uses MIDP 2.0.

3. Building and running

The example is built using the standard method for building and running as described in Building and running the examples.

The following classes should be included in the jar:

com.nokia.midp.examples.security.pki.*

No resource files are used.

When launched, the application displays the following screen.

4. Design and implementation

Application classes

  • PKIMIDlet A simple MIDlet generating signatures using CMSMessageSignatureService-class.

System classes

  • Form A Screen that contains a mixture of Items, and handles layout and user interaction focus.
  • Display The manager of the display and input devices of the system.

5. Summary

This example provids the source code to demonstrate how to digitally sign data and manage user certificates.
